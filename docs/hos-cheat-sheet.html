§1 Core HOS Concepts

Regulator & Scope

  • FMCSA (Federal Motor Carrier Safety Administration) — part of U.S. DOT.
  • Rules apply to property-carrying CMVs in interstate commerce under 49 CFR Part 395.
  • Assessment assumption: property-carrying driver, 70-hr / 8-day schedule, no adverse driving conditions.

Commercial Motor Vehicle (CMV)

A CMV is a truck/truck-tractor with trailer used in interstate commerce that meets any of:

  • Weighs (incl. load) ≥ 10,001 lbs
  • GVWR or GCWR ≥ 10,001 lbs
  • Transports placarded hazardous materials

Key formulas:

  • On-duty time = Driving + On Duty Not Driving
  • Off-duty time = Off Duty + Sleeper Berth
  • 70-hr / 8-day limit applies to total on-duty time (not just driving)

§3 Daily Limits (Workday Rules)

Rest before workday
≥ 10 consecutive hours
Off Duty and/or Sleeper Berth — no on-duty or driving
14-Hour Driving Window
14 consecutive hours
Starts when driver goes on-duty after 10-hr break. May drive ≤ 11 hrs within it. After 14th hour: may not drive again until new rest.
11-Hour Driving Limit
≤ 11 hours driving
Max driving time within the 14-hr window. Once reached, no more driving until next qualifying rest.
30-Minute Break
≥ 30 consecutive min
Required after 8 cumulative hours of Driving (since last break). Can be OFF, SB, or ON — must be 30 continuous min without Driving. Does not extend 14-hr window or 11-hr max.
Common trap: The 30-min break must be taken before the 8th driving hour is reached. Once 8 cumulative driving hours elapse without a break, the driver has violated the rule.

§4 Weekly / Cycle Limits (70-Hour / 8-Day Rule)

4.1 Rolling 8-Day Window

  • Sum on-duty hours for current day + previous 7 days (8 total).
  • If total ≥ 70 → driver may NOT drive (may still do on-duty not driving work).
  • Drop the day that is 9 days ago as each new day starts.
  • Assessment input: Current Cycle Used (Hrs) — pre-filled on-duty hours already consumed.

4.2 34-Hour Restart

  • ≥ 34 consecutive hours off-duty (OFF + SB) → resets the 70-hr accumulation to zero.
  • Optional but important for trip planning when cycle is near 70.
  • A new 8-day cycle begins after restart.
Implementation tip: Maintain a rolling array of last 8 days' on-duty totals. When the sum approaches 70, schedule a 34-hr restart period.

§5 Sleeper Berth Provisions

5.1 Simple Case (Initial Implementation)

≥ 10 consecutive hours in Sleeper Berth (or combined off-duty) → fully resets 11-hr driving limit and 14-hr window.

5.2 Split Sleeper (Future / Advanced)

  • Two qualifying periods: e.g., 7 hrs SB + 3 hrs OFF (total ≥ 10 hrs).
  • Paired periods may not count against the 14-hr window in certain calculations.
  • The FMCSA guide contains detailed recalculation examples.
Assessment scope: Model only "10 consecutive hours off" for initial implementation. Split sleeper can be an extension later.

§7 Driver's Daily Log (Record of Duty Status)

7.1 Grid Structure

  • 24-hr horizontal axis from midnight (00:00) to midnight (24:00).
  • 4 horizontal rows: Off Duty, Sleeper Berth, Driving, On Duty Not Driving.
  • Continuous line segments in the appropriate row + vertical lines at each status change.
  • Total time across all rows = 24 hours.

7.2 Required Information per Log

  • Date (month, day, year)
  • Total miles driven
  • Truck/tractor & trailer numbers
  • Carrier name & main office address
  • Driver's full signature
  • Co-driver name (if any)
  • Terminal time zone & 24-hr starting time
  • Remarks with city/state for each status change
  • Shipping document numbers or shipper/commodity
  • Special notes (e.g., adverse conditions)

In the app: Generate one or more daily logs per trip, displayed visually in the frontend as 24-hr grids.

§9 Algorithm & Implementation Plan

9.1 Scheduling Algorithm (Pseudo-code)

FUNCTION plan_trip(current_loc, pickup, dropoff, cycle_used):
  drive1 = distance(current_loc → pickup) / avg_speed   # hours driving
  drive2 = distance(pickup → dropoff) / avg_speed

  days = []
  day = new_day(cycle_used)
  on_duty_today = 0; driving_today = 0; window_start = None

  # --- Leg 1: Current → Pickup ---
  schedule_leg(drive1, day, loading=0)
  # Add 1 hr ON for pickup loading
  schedule_task(1, "on_duty", day)

  # --- Leg 2: Pickup → Dropoff ---
  schedule_leg(drive2, day, loading=1)

  RETURN days

FUNCTION schedule_leg(drive_hours, day, loading):
  DRIVING_PER_DAY = 11
  WINDOW = 14
  BREAK_AFTER = 8

  while drive_hours > 0:
    # Start new day if needed
    if day.full() or day.cycle_full():
      day = new_day()

    # Start 14-hr window
    if day.window_active is False:
      day.start_window()

    # 30-min break check
    if day.driving_since_break >= BREAK_AFTER:
      schedule_break(0.5, day)

    # Drive up to remaining allowed hours
    allowed_drive = min(drive_hours,
                        DRIVING_PER_DAY - day.driving_today,
                        WINDOW - day.on_duty_since_window_start)
    schedule_drive(allowed_drive, day)
    drive_hours -= allowed_drive

    # Add loading/unloading ON time
    if loading:
      schedule_task(1, "on_duty", day)

  return day

9.2 Route & Location Strategy

  • Recommended: Use OpenRouteService (free, no API key for limited use) or OSRM demo server.
  • Fallback: Great-circle distance × 1.3 (road factor) ÷ avg_speed (55 mph).
  • Annotate route points → assign locations to each log status change.
